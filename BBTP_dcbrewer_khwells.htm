Project 2 Black Box Test Plan

 

Document Author(s): Daniel Brewer, Kaleb Wells

 

Date: Monday, July 27, 2015

 

 

Introduction

 

There are 3 test cases below designed to test the “edge cases” of the program. Throughout all of the test cases, the tester is expected to watch the buttons and text to make sure it is displayed correctly. As the project description provides limited detail on button behavior, the following button behavior was implemented and is expected:

 

  • The user should not be able to click “next question” unless they have already used all of their attempts for the given question
  • The user should not be able to click “submit answer” unless they have selected one of the answer choices
  • If a user submits an incorrect answer, they should not be able to submit the same answer again

 

The file used for the test cases is “test.xml”. It contains 9 questions; 3 from each category. Each question title contains the question’s difficulty and the correct answer, to make testing easier. For example, “Easy Question (Pick A)” is the first elementary question in the file. This file can be loaded with the file loading window from starting the program, or from a command line argument passed into the program.

 

Note: every question has the same answer choices, [A-1, B-2, C-3, D-4]

 

Test case 1: Quit immediately

Preconditions: Program is freshly started, “test.xml” is loaded.

Procedure: Select the “Quit” button

Expected behavior: Quiz window closes, performance report comes up saying “0 Questions correct out of 0 attempts”

 

Note:

Tests cases 2 and 3 have long, formulaic preconditions and procedures. Because of this, they are organized into tables. Each column is explained below:

 

Answer submitted: The answer the tester should submit to the given question

 

Question text: The text of the question, e.g. “What was the most successful film of 2010?”

 

Choice A, B, C or D: Which answer the tester should submit to the given question

 

Response text: What response is expected to the tester’s answer submission.

 

Correct button behavior?: Whether or not the buttons behaved correctly for this test.

 

Met expectations?: Whether or not the interface’s text was correct.
